As pessoas não devem acreditar que os modelos de IA sabem realmente de tudo. Algo que acontece muito atualmente é o fato das pessoas perguntarem tudo para a IA e acreditarem que ela tem a resposta para tudo. E ela não tem — ela mal tem informações sobre ela mesma.

Como assim? Se você perguntar para algum modelo qual a melhor forma de usar o Claude Code, pode ter certeza absoluta que ele em momento algum vai te contar sobre Context Window, não vai saber explicar como utilizar /goal da melhor forma (talvez nem sequer cite sobre), não vai explicar sobre a dumbzone, não vai explicar sobre as skills que deve aprender (no máximo vai falar sobre as skills que veem no próprio Claude Code).

A LLM não vai te ensinar as boas práticas, até porque ela não sabe. Se ela soubesse as boas práticas de tudo, não iria gerar tanto *AI slop*.

É interessante que muitas vezes, se você perguntar sobre qual stack ou linguagem de programação usar para desenvolver uma aplicação, a LLM não vai saber qual a melhor stack para ela mesma. Possivelmente vai dizer que linguagens como Rust são difíceis e não vai recomendar.

Isso é irônico, pois existe um benchmark feito por Kun Chen (Member of the Technical Community, autor de https://axi.md, ex-engenheiro L8 na Meta, Microsoft e Atlassian) onde ele testou exatamente isso: [Evaluating the Effectiveness of Programming](https://blog.kunchenguid.com/p/evaluating-the-effectiveness-of-programming).

![Easy Tasks](/images/Easy.png)

Em tarefas fáceis, Rust foi a melhor entre todas as testadas. Como que para fazer uma tarefa fácil, Rust é melhor que JavaScript ou Python, que são tão simples?

![Hard Tasks](/images/Hard.png)

Isso não quer dizer que Rust seja a melhor linguagem. No mesmo teste, Rust teve a maior queda quando o teste foi feito em tarefas difíceis.
